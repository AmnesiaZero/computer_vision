Основные операции в OpenCV для работы с изображениями связаны с открытием и сохранением файла с изображением, а также с масштабированием, поворотом, кадрированием и отражением по осям. Для открытия файла с изображением нам необходимо знать две функции: cv2.imread() cv2.imshow(). Функция `cv2.imread()` осуществляет чтение и загрузку в оперативную память массива пиксельных данных, представляющих растровое изображение, из указанного файла. Аргументом функции является строка, содержащая имя файла или полный путь к файлу. Возвращаемое значение функции — многомерный массив NumPy , содержащий пиксельные данные изображения. Функция `cv2.imshow()` отвечает за визуализацию растрового изображения на экране пользователя. Она принимает два аргумента: идентификатор окна отображения (строка) и ссылку на многомерный массив NumPy , содержащий пиксельные данные изображения, предварительно загруженные функцией `cv2.imread()`. import cv2 #подключение библиотеки img =cv2.imread("image.jpg") # чтение изображения в пере м енную img cv2.imshow(" Show ", img ) # Отображение изображения в новом окне с подписью Show cv2.waitKey(0) # Ожидание нажа т ия клавиши cv2.destroyAllWindows() # Для корректного закрытия окон Изменение размера изображения в OpenCV выполняется с помощью функции cv2.resize(). В библиотеке OpenCV масштабирование растрового изображения реализуется функцией `cv2.resize()`, принимающей три аргумента и возвращающей модифицированное изображение. Первый аргумент представляет собой NumPy -массив, содержащий пиксельные данные исходного изображения. Второй аргумент — это упорядоченный набор (кортеж) из двух целых чисел, определяющих новые размеры изображения (ширина, высота) в пикселях. Третий аргумент задаёт метод интерполяции, используемый для пересчёта пиксельных значений при изменении размеров. Кортеж, в данном контексте, представляет собой неизменяемый объект, содержащий пару значений, определяющих геометрические параметры выходного изображения. В отличие от списков Python , кортежи обладают свойством неизменяемости, что обеспечивает целостность параметров масштабирования. import cv2 wide = 250 img = cv2.imread("image.jpg") #вычисляем коэффициент f , чтобы сохранить соотношение сторон f = float( wide) / img.shape [1] # формируем кортеж ( x , y ) с размерами изображения new_size = (wide, int ( img.shape [0] * f)) # изменяем изображение и возвращаем его в переменную res res = cv2.resize( img , new_size , interpolation = cv2.INTER_AREA) cv2.imshow( "Original", img ) cv2.imshow( "Resize", res) cv 2. waitKey ( 0) cv 2. destroyAllWindows () Логика работы программы понятна из комментариев. Изменяя параметр wide , можно уменьшать или увеличивать размер конечного изображения. При формировании кортежа с размерами изображения применяется команда img.shape [0]. Команда img.shape [i] возвращает ширину (при i = 1) или высоту (при i = 0) исходного изображения img . Интерполяция — это метод, который OpenCV использует для вычисления значений пикселей при изменении размера изображения. В качеств значения этого параметра могут быть указаны значения: cv2.INTER_AREA, cv2.INTER_NEAREST, cv2.INTER_LINEAR, cv2.INTER_CUBIC, и cv2.INTER_LANCZOS4. Метод cv2.INTER_AREA (интерполяция по области) разработан специально для уменьшения размера изображения. Он вычисляет значение каждого пикселя в уменьшенном изображении на основе среднего значения пикселей из соответствующей области в исходном изображении. Метод `cv2.INTER_NEAREST` (интерполяция по ближайшему соседу) разработан как самый простой и быстрый метод интерполяции. Он присваивает значение пикселя в выходном изображении, беря значение цвета пикселя из ближайшего пикселя в исходном изображении, без каких-либо вычислений или "смешивания". Этот метод наименее подходит как для уменьшения, так и для увеличения изображения в большинстве задач, но может использоваться при масштабировании пиксельной графики, где не нужна гладкость. Метод `cv2.INTER_LINEAR` (билинейная интерполяция) разработан для создания баланса между скоростью и качеством. Он вычисляет значение пикселя в выходном изображении, используя линейную интерполяцию, основанную на значениях 2x2 окрестности пикселей в исходном изображении, проводя "смешивание" значений соседних пикселей. Этот метод хорошо подходит как для умеренного уменьшения, так и для умеренного увеличения изображения. Метод `cv2.INTER_CUBIC` (бикубическая интерполяция) разработан для достижения более высокого качества изображения, чем `cv2.INTER_LINEAR`. Он вычисляет значение пикселя в выходном изображении, используя кубическую интерполяцию, основанную на значениях 4x4 окрестности пикселей в исходном изображении, что позволяет получить более точное "смешивание" цветов, но требует больше вычислительных ресурсов. Этот метод хорошо подходит для увеличения изображения, когда нужно сохранить высокое качество, но также может использоваться и для уменьшения (хотя не является лучшим выбором для этого). Метод `cv2.INTER_LANCZOS4` (интерполяция Ланцоша ) разработан для обеспечения наилучшего качества изображения, особенно при увеличении. Он вычисляет значение пикселя в выходном изображении, используя более сложную функцию Ланцоша на основе значений 8x8 окрестности пикселей в исходном изображении, что позволяет сохранить детали, резкость и избежать размытия, но является самым медленным методом. Этот метод лучше всего подходит для увеличения изображения, когда требуется максимально высокое качество и сохранение резкости, но может быть использован и для уменьшения, хотя и не является оптимальным выбором из-за вычислительной стоимости. Вырезка фрагмента изображения В задачах компьютерного зрения часто возникает необходимость сегментации изображения, заключающаяся в выделении области интереса (ROI — Region Of Interest ) для последующего анализа. Это достигается путём исключения из рассмотрения нерелевантных фрагментов изображения, что выполняется операцией обрезки ( cropping ). Операция обрезки реализуется посредством механизма срезов ( slicing ), представляющего собой выбор подмножества элементов из многомерного массива данных, представляющего изображение. Изображение моделируется как двумерный массив пикселей с координатами (x, y), где (0, 0) соответствует левому верхнему углу, а (N-1, M-1) — правому нижнему (N — ширина, M — высота). Обрезка осуществляется путём задания ограничивающих координат (x1, y1) и (x2, y2), определяющих прямоугольную область, которая затем извлекается из исходного массива данных.
