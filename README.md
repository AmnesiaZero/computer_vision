# Computer Vision Practices

Этот репозиторий содержит практические работы по компьютерному зрению в папках `1`–`10`, а также дополнительные варианты в `5idk` и `wathersheld`.

- `cd 1 && streamlit run main.py`
- `cd 2 && python main.py`
- `cd 3 && python main.py`
- `cd 4 && python main.py`
- `cd 5 && python main.py` (и отдельно `python split_merge_segmentation.py`)
- `cd 6 && python main.py`
- `cd 7 && python main.py`
- `cd 8 && python main.py`
- `cd 9 && python main.py`
- `cd 10 && python main.py`

## Что происходит в каждой практике

### Практика 1 (`1/main.py`)
- Это интерактивный редактор на Streamlit.
- Загружает изображение, применяет операции обработки (изменение размера, фильтры, поворот, сегментация и т.д. в зависимости от доступных контролов UI).
- Пользователь работает через веб-интерфейс, сразу видит результат.

### Практика 2 (`2/main.py`)
- Загружает изображение и переводит его в HSV.
- Строит бинарную маску только для жёлтого диапазона.
- Для жёлтой маски ищет контуры, вычисляет центр(ы) масс и рисует координаты на изображении.
- Окна OpenCV показывают исходник, `yellow_mask` и итог детекции.

### Практика 3 (`3/main.py`)
- Загружает изображение и применяет детектор углов Харриса.
- Находит «угловые» точки по матрице отклика.
- Подсвечивает найденные углы красным на изображении.

### Практика 4 (`4/main.py`)
- Демонстрирует базовые методы сегментации/бинаризации.
- Показывает: global threshold, adaptive threshold, Otsu, Canny.
- На выходе — набор сравнительных изображений по каждому методу.

### Практика 5 (`5/main.py` и `5/split_merge_segmentation.py`)
- `main.py`: сегментация Region Growing (рост области от seed-точки по порогу похожести).
- `split_merge_segmentation.py`: рекурсивное деление и объединение регионов по однородности.
- Выход — бинарная маска или усредненная сегментированная карта.

### Практика 6 (`6/main.py`)
- Выполняет сегментацию изображения методом Mean Shift.
- Сглаживает цветовые области и группирует близкие пиксели.
- Показывает сегментированное изображение (плюс есть демо K-Means в отдельном скрипте).

### Практика 7 (`7/main.py`)
- Генерирует синтетические точки 3 классов.
- Обучает k-NN и оценивает точность.
- Строит scatter-плот и карту границ решений классификатора.

### Практика 8 (`8/main.py`)
- Загружает MNIST.
- Строит MLP (полносвязную нейросеть) для классификации цифр.
- Обучает модель, печатает итоговые метрики, показывает графики loss/accuracy.

### Практика 9 (`9/main.py`)
- Загружает MNIST.
- Строит CNN (сверточную нейросеть).
- Обучает, валидирует и оценивает качество на тестовой выборке.

### Практика 10 (`10/main.py`)
- Загружает CIFAR-100.
- Запускает сравнительное обучение нескольких архитектур (VGG-like, ResNet-like, DenseNet-like).
- Сравнивает модели по метрикам и числу параметров.

### Дополнительно: `wathersheld/main.py`
- Классическая сегментация методом водораздела.
- Показ промежуточных этапов: threshold, foreground/background, markers, итог.

### Дополнительно: `5idk/main.py`
- Еще один вариант водораздела с визуализацией шагов.
- Удобен для отчета, где нужно показать pipeline по этапам.

## Основные термины и определения

- **Компьютерное зрение (Computer Vision)** — методы, позволяющие алгоритмам анализировать изображения и видео.
- **Изображение в оттенках серого (Grayscale)** — одно канальное изображение яркости без цвета.
- **Цветовое пространство BGR/RGB** — представление цвета через каналы синего, зеленого, красного.
- **Цветовое пространство HSV** — Hue (тон), Saturation (насыщенность), Value (яркость).
- **Сегментация** — разбиение изображения на области/объекты по выбранному критерию.
- **Бинаризация (Thresholding)** — превращение изображения в черно-белое по порогу.
- **Global Threshold** — один общий порог для всего изображения.
- **Adaptive Threshold** — локальный порог для каждой области изображения.
- **Метод Оцу (Otsu)** — автоматический выбор порога по гистограмме.
- **Контур (Contour)** — граница найденного объекта на бинарной маске.
- **Моменты изображения** — числовые характеристики формы; по ним можно вычислить центр масс.
- **Region Growing** — рост области от стартового пикселя, добавляя похожие соседние пиксели.
- **Split-and-Merge** — рекурсивное деление изображения на квадранты и объединение однородных частей.
- **Watershed (Водораздел)** — разделение соприкасающихся объектов через маркеры и «топографический» подход.
- **K-Means** — кластеризация в `k` групп по близости признаков.
- **Mean Shift** — кластеризация без фиксированного `k`, сдвиг к максимумам плотности.
- **Детектор Харриса** — алгоритм поиска углов (характерных точек) в изображении.
- **Градиент/Край (Edge)** — резкое изменение яркости; основа для выделения границ.
- **k-NN (k ближайших соседей)** — классификатор, голосующий по ближайшим примерам.
- **Train/Test split** — разделение данных на обучение и проверку.
- **MLP** — полносвязная нейросеть (вход -> скрытые слои -> выход).
- **CNN** — сверточная нейросеть, извлекающая локальные признаки фильтрами.
- **Эпоха (Epoch)** — один полный проход по обучающему набору.
- **Batch size** — число примеров, обрабатываемых за один шаг обновления весов.
- **Loss (функция потерь)** — мера ошибки модели.
- **Accuracy** — доля правильных предсказаний.
- **Overfitting (переобучение)** — модель хорошо запоминает train, но плохо обобщает на test.

## Примечания

- Основной файл запуска в каждой практике — `main.py`.
- Дополнительные/тестовые сценарии вынесены в отдельные скрипты внутри папок.
- Изображения имеют префикс `img_` и лежат рядом со скриптами.
